Using laravel-websockets
First, you need to have installed the websocket package (This package has been tested using laravel-websockets v1.12).
Ideally, you should follow the official Installation Documentation from beyondcode if you are doing a fresh installation.
composer require beyondcode/laravel-websockets "^1.12"
Once you have installed and configured the websocket package, set your .env variables and update the default pusher config:
Updated broadcasting.php config per beyondcode's documentation

'pusher' => [
'driver' => 'pusher',
'key' => env('PUSHER_APP_KEY'),
'secret' => env('PUSHER_APP_SECRET'),
'app_id' => env('PUSHER_APP_ID'),
'options' => [
'cluster' => env('PUSHER_APP_CLUSTER'),
'encrypted' => true,
'host' => 'localhost',
'port' => 6001,
'scheme' => 'http'
],
],

BROADCAST_DRIVER=pusher

Having recently built a new server for a client (which is required to run Windows Server OS) using an ASRock Z97 Extreme4 motherboard, I ran into some frustrating issues:

•The built-in Intel I218V ethernet adapter had not been installed, and refused to be installed by conventional means


•Running Intel's own ethernet installer package froze-up the system


After spending an hour reading about .INF hacks, custom-made drivers and complaints aplenty, I began some trial-and-error with the various .INF files provided from Intel's website. Lo-and-behold, one works!

SOLUTION:

•Download Intel's PROWinx64.exe driver package: https://downloadcenter.intel.com/Detail_Desc.aspx?DwnldID=23071&lang=eng&ProdId=3707


•You'll need an archive utility such as 7zip or WinRAR so that you can manually decompress thePROWinx64.exe file. (Note: PROWinx64.exe will self-extract if you run it, however, upon exiting it deletes all of the resulting files, which doesn't help us out a lot.)

Right-click on the PROWinx64.exe file and choose "Extract to..." or "Extract to PROWinx64" from within the 7zip or WinRAR menu. This will decompress the file to a folder.


•Make note of where the resulting folder is located on your computer. (i.e. Mine ended up being thePROWinx64 folder on my desktop.)


•Open the Device Manager (i.e. Run > mmc devmgmt.msc) and locate the "Ethernet Adapter" or "Unknown Ethernet Adapter" device.

Right-click it and choose "Update Driver Software..."


•In the box which appears, choose Browse my computer for driver software > Let me pick from a list of device drivers on my computer > Show All Devices > Have Disk...

Navigate to the folder where PROWinx64 extracted to.


•Inside the PROWinx64 folder, navigate to PRO1000 > Winx64 > NDIS64 and choose any of the .INF files.


•A list of available drivers will appear: Scroll down until you see the one listed as Intel(R) Ethernet Connection I218-LM - Click that, then click "Next"

(NOTE: There is also a listing for Intel(R) Ethernet Connection (2) I218-LM - This didn't work in my tests, so I'd advise ensuring you avoid that one.)


•An "Update Driver Warning" message will appear informing you that the driver isn't made for Server 2012. Simply click "Yes" to continue.


If you've made it this far, the i218V should install successfully, and you should be able to connect to your LAN network and Internet as well.
